# Chapter 4 – Software (IGCSE Computer Science)

## 4.1 Types of Software and Interrupts

### 4.1.1 System Software and Application Software

Software is divided into two main categories:

#### System Software

System software manages and controls computer hardware and provides a platform for application software to run.

Characteristics:

* Controls and manages hardware
* Provides Human Computer Interface (HCI)
* Allocates hardware resources
* Allows hardware and software to work together
* Provides a platform for applications

Examples:

* Operating Systems
* Utility Software
* Device Drivers
* Firmware

#### Application Software

Application software enables users to perform specific tasks.

Examples:

* Word Processors
* Spreadsheets
* Databases
* Web Browsers
* CAD Software
* Games

---

### 4.1.2 Utility Software

Utility software helps maintain, manage and protect a computer system.

Types:

#### Anti-virus Software

Functions:

* Detects malware
* Removes malware
* Quarantines infected files
* Scans storage devices
* Updates virus definitions

#### Backup Software

Purpose:

* Creates copies of files
* Restores lost data
* Protects against hardware failure

Methods:

* Full Backup
* Incremental Backup
* Differential Backup

#### Data Compression Software

##### Lossless Compression

* No data lost
* Original file recovered exactly
* Example: ZIP

##### Lossy Compression

* Some data permanently removed
* Smaller file size
* Example: JPEG, MP3

#### Disk Defragmentation

* Rearranges fragmented files
* Improves HDD performance
* Not normally required for SSDs

---

### 4.1.3 Operating Systems

An Operating System (OS) is system software that manages computer resources.

Examples:

* Windows
* Linux
* macOS
* Android
* iOS

Functions of an Operating System:

#### Memory Management

* Allocates RAM
* Frees unused memory
* Uses virtual memory

#### Processor Management

* Allocates CPU time
* Schedules processes
* Handles multitasking

#### Security Management

* User accounts
* Passwords
* Access rights
* Data protection

#### Hardware Peripheral Management

* Controls input/output devices
* Uses device drivers
* Manages buffers and queues
* Handles printer communication

#### File Management

* Create files
* Delete files
* Rename files
* Move files
* Copy files
* Manage folders/directories
* Control file permissions

#### Multitasking

Allows multiple processes to run simultaneously.

Features:

* Process scheduling
* Priority allocation
* Resource sharing
* Interrupt handling

---

### 4.1.4 Firmware

Firmware is software permanently stored in ROM or flash memory.

Characteristics:

* Controls hardware directly
* Starts before the operating system
* Usually difficult to modify

Examples:

* BIOS
* Embedded system firmware
* Router firmware

---

### Running Application Software

Application software requires:

1. Hardware
2. Firmware
3. Operating System

Execution sequence:

1. Firmware starts computer
2. Operating System loads
3. Application software runs

---

### Interrupts

An interrupt is a signal sent to the CPU requesting immediate attention.

Purpose:

* Suspend current process
* Handle urgent events
* Resume previous task afterward

#### Hardware Interrupts

Generated by:

* Keyboard
* Mouse
* Printer
* Disk drives

Examples:

* Key pressed
* Printer out of paper

#### Software Interrupts

Generated by:

* Operating system
* Applications

Examples:

* Division by zero
* Invalid instruction

Interrupt Process:

1. Interrupt occurs
2. CPU finishes current instruction
3. Current state saved
4. Interrupt Service Routine (ISR) executed
5. Previous state restored
6. Program continues

---

## 4.2 Programming Languages, Translators and IDEs

### 4.2.1 High-Level Languages

Characteristics:

* Human-readable
* Easier to learn
* Portable across systems
* Faster development

Examples:

* Python
* Java
* Visual Basic
* C++

Advantages:

* Easier debugging
* Easier maintenance
* Faster development

Disadvantages:

* Requires translation
* Less hardware control

---

### 4.2.2 Low-Level Languages

Characteristics:

* Closer to machine code
* Hardware specific

Types:

#### Machine Code

* Binary instructions
* Directly executed by CPU

Advantages:

* Fast execution
* No translator required

Disadvantages:

* Difficult to write
* Difficult to debug

#### Assembly Language

Characteristics:

* Uses mnemonics
* Easier than machine code

Examples:

* ADD
* MOV
* SUB
* JMP

Requires:

* Assembler

Advantages:

* Fast execution
* Hardware control

Disadvantages:

* Machine dependent
* Difficult compared with high-level languages

---

### 4.2.3 Translators

Translators convert source code into machine code.

Three Types:

#### Assembler

Input:

* Assembly language

Output:

* Machine code

#### Compiler

Process:

* Translates entire program at once

Output:

* Executable file

Advantages:

* Faster execution
* Executable can run without compiler
* Less memory usage

Disadvantages:

* Longer development cycle
* Must recompile after changes

#### Interpreter

Process:

* Translates one line at a time

Advantages:

* Easier debugging
* Immediate error reporting
* Faster testing

Disadvantages:

* Slower execution
* Interpreter required to run program

---

### Compiler vs Interpreter

| Feature           | Compiler       | Interpreter        |
| ----------------- | -------------- | ------------------ |
| Translation       | Entire program | One line at a time |
| Execution Speed   | Faster         | Slower             |
| Debugging         | Harder         | Easier             |
| Executable File   | Yes            | No                 |
| Development Speed | Slower         | Faster             |

---

## 4.2.5 Integrated Development Environment (IDE)

An IDE is software used to develop programs.

Examples:

* PyCharm
* IDLE
* Visual Studio
* BlueJ

### Features of IDEs

#### Code Editor

* Write code
* Edit code
* Syntax highlighting

#### Translator

* Compiler and/or Interpreter

#### Runtime Environment

* Executes programs

#### Debugger

Functions:

* Breakpoints
* Step-through execution
* Variable inspection
* Error tracing

#### Error Diagnostics

* Detect syntax errors
* Highlight mistakes
* Suggest corrections

#### Auto-completion

* Predicts code
* Speeds up typing

#### Auto-correction

* Fixes common mistakes

#### Auto-documentation

* Generates documentation automatically

#### Pretty Printing

* Improves code formatting
* Enhances readability
